# 논문 지도 시스템 프로토타입 범위 정의

## 문서 작성 일자
2025-12-08 (수정)

## 목적
프로토타입 개발을 위해 **가시적으로 확인 가능한 화면 및 비즈니스 로직**만 추출하여 정리
- ERP 연동, SSO, 외부 시스템 연동 제외
- Mock 데이터를 활용한 화면 구현 중심
- 프론트엔드 UI/UX 및 클라이언트 로직에 집중

## 프로토타입 구현 범위

### 1. 사용자 상태 표시 및 안내 화면

#### 1.1 관리자 화면
- **사용자 목록 화면** (Mock 데이터 기반)

  - [ ] 검색 기능 (이름, 학번/사번)
  - Mock 데이터: 학생 20명, 교수 10명, 직원 5명

#### 1.2 교수 화면
- **대시보드 안내 화면**

#### 1.3 학생 화면

  - [ ] 배정 완료 시 지도교수 정보 카드 표시

#### 2.3 학생 화면

### 3. 지도교수 배정

**메뉴명 변경**: 연구계획서 관리 → **지도교수 배정**
**프로세스**: 학생이 연구계획서 제출 → 관리자가 연구계획서를 확인하고 지도교수 매핑

#### 3.1 관리자 화면
- **지도교수 배정 화면** (신규 구현)
  - [ ] 학생 목록 테이블
    - 학번, 이름, 학과, 연구계획서 제출일, 배정 상태, 연구계획서 보기
  - [ ] 연구계획서 보기 모달
    - 학생 정보, 연구 주제, 연구 목적, 연구 방법
  - [ ] 행 클릭 → 교수 배정 모달
  - [ ] 주지도교수 검색 모달
    - 이름/사번 검색
    - 소속 학과 필터 (우선 표시)
    - 타과 포함 검색 체크박스
    - 전임교원 우선 정렬 (배지 표시)
    - 라디오 버튼으로 1명 선택
  - [ ] 공동지도교수 검색 모달
    - 동일 검색 UI
    - 체크박스로 N명 선택
  - [ ] 배정 완료 버튼 클릭 → 상태 업데이트
  - Mock 데이터: 학생 20명, 연구계획서 데이터, 교수 10명 (소속 학과 5개)

#### 3.2 학생 화면
- **연구계획서 제출 화면**
  - [ ] 제출 후 "배정 대기 중" 상태 표시
  - [ ] 배정 완료 시 알림 메시지 및 지도교수 정보 표시

### 4. 논문 지도 단계 관리

**명칭 일원화**: Workflow → **논문 지도 단계**

#### 4.1 관리자 화면
- **논문 지도 단계 설정 화면** ⭐ **(신규 추가 - 기존 프로토타입에서 누락)**
  - **메뉴 위치**: 시스템 설정 > 논문 지도 단계 설정
  - [ ] 논문 지도 단계 목록 테이블
    - 단계명, 학위 구분 (석사/박사), 단계 수, 등록일, 수정 버튼
  - [ ] 학위별 단계 템플릿 관리
    - 논문 지도 단계명 (예: "석사 표준 과정", "박사 표준 과정")
    - 단계 목록 (순서, 단계명, 유형: 제출/심사)
  - [ ] 단계 추가/수정/삭제 UI
  - [ ] 드래그 앤 드롭으로 순서 변경
  - [ ] 평가표 적용 여부 드롭다운
  - Mock 데이터: 2개 논문 지도 단계 (석사 5단계, 박사 7단계)

- **학생 단계 관리 화면** (신규 구현)
  - [ ] 학생 목록 테이블
    - 학번, 이름, 학과, 현재 단계, 논문 지도 단계
  - [ ] 필터: 학과, 학년, 논문 지도 단계
  - [ ] 체크박스 선택 + 논문 지도 단계 드롭다운 + "일괄 부여" 버튼
  - [ ] "다음 단계로 이관" 버튼
    - 클릭 시 조건 체크 (심사 완료 여부)
    - 조건 미충족 시 경고 메시지
  - Mock 데이터: 학생 20명, 각 단계별 분포

#### 4.2 교수 화면
- **학생 단계 조회**
  - [ ] 지도 학생 목록에서 현재 단계 표시 (읽기 전용)

#### 4.3 학생 화면
- **대시보드**
  - [ ] 현재 단계 진행 상황 카드
    - 단계명, 설명, 완료 여부
  - [ ] 다음 단계 안내 메시지
  - [ ] 진행률 표시 (프로그레스 바)

### 5. 심사 및 평가 관리

#### 5.1 관리자 화면
- **심사 기준 등록 화면** (기존 확인 후 보완)
  - [ ] 평가표 유형 선택 라디오 버튼
    - 점수형 (100점 만점)
    - P/F형 (합격/불합격)
  - [ ] 평가 항목 관리
    - 항목명, 배점 (점수형만), 설명
    - 추가/수정/삭제 버튼
  - Mock 데이터: 점수형 평가표 1개 (5개 항목), P/F형 평가표 1개

- **심사위원 배정 화면** (신규 구현)
  - [ ] 심사 대상 학생 목록 테이블
    - 학번, 이름, 학과, 심사 단계, 심사위원 배정 상태
  - [ ] 행 클릭 → 심사위원 배정 모달
  - [ ] 심사위원장 검색
    - 라디오 버튼으로 1명 선택
    - 이름/사번 검색
  - [ ] 심사위원 검색
    - 체크박스로 N명 선택
    - 지도교수 자동 포함 표시
    - 석사: 3명, 박사: 5명 가이드 메시지
  - [ ] 일괄 저장 버튼
  - Mock 데이터: 학생 10명, 교수 10명

- **심사 결과 조회/판정 화면**
  - [ ] 심사 대상 학생 목록
  - [ ] 행 클릭 → 심사 결과 상세 모달
  - [ ] 심사위원별 점수 입력 현황 테이블
    - 심사위원명, 점수, 제출일, 상태 (미제출/제출 완료)
  - [ ] 자동 판정 로직 (프론트엔드)
    - 석사: 3명 중 2명 이상 70점 이상 → 통과
    - 박사: 5명 중 3명 이상 70점 이상 → 통과
  - [ ] 최종 판정 결과 표시
    - 통과, 보완 후 통과, 보류, 불합격
  - [ ] 심사위원장 최종 승인 버튼
  - Mock 데이터: 심사 완료 5건, 진행 중 3건

#### 5.2 교수 화면
- **심사위원 점수 입력 화면** (신규 구현)
  - [ ] 본인이 심사위원인 학생 목록
    - 학번, 이름, 학과, 심사 단계, 제출 상태

  - Mock 데이터: 심사 대상 학생 3명

- **최종 판정 화면** (심사위원장용)
  - [ ] 모든 심사위원 점수 확인 테이블
  - [ ] 자동 판정 결과 확인
  - [ ] 최종 판정 승인 버튼

#### 5.3 학생 화면

## 프로토타입 우선순위

### P1 (필수 - 1차 구현)
1. **지도교수 배정 화면** (관리자) ⭐ 메뉴명 변경
   - 연구계획서 보기 모달, 교수 검색 모달, 배정 처리
2. **논문 지도 단계 설정 화면** (관리자) ⭐ 신규 추가
   - 학위별 단계 템플릿 관리, 드래그 앤 드롭 순서 변경
3. **학생 단계 관리** (관리자)
   - 논문 지도 단계 부여, 단계 이관
4. **심사위원 배정 UI** (관리자)
   - 심사위원 검색 모달, 배정 처리
5. **심사 점수 입력 화면** (교수)
   - 평가 항목 입력, 자동 계산
6. **사용자 상태 안내 화면** (교수/학생)
   - 미배정 시 안내 메시지

### P2 (중요 - 2차 구현)
1. **권한 관리 UI** (관리자)
   - 메뉴 접근 권한 매트릭스
2. **사용자 지원 (대리 로그인)** (관리자)
   - 사용자 검색, 화면 전환
3. **심사 결과 판정 화면** (관리자)
   - 자동 판정 로직, 최종 승인
4. **메뉴 동적 제어** (교수/학생)
   - 상태별 메뉴 노출/숨김
5. **일정 조회 화면** (교수/학생)
   - 캘린더 뷰, 기간 안내

### P3 (개선 - 3차 구현)
1. **평가표 구성 UI** (관리자)
   - 점수형/P/F형 선택
2. **테이블 UI 개선** (공통)
   - 정렬, 페이지네이션, 반응형
3. **로딩 상태 표시** (공통)
   - 스켈레톤 UI
