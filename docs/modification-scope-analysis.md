## 1. 로그인 및 사용자 동기화

### 1.1 관리자 화면
#### 신규 구현 필요
- **사용자 관리 화면**
  - 현재 상태: 메뉴 존재 (`userManagement`)

    - [ ] ERP 연동 사용자 동기화 버튼 추가

    - [ ] 사용자 목록 조회 (학생/교수/직원 구분)
    - [ ] 사용자별 상태 표시 (논문과정 선택 여부)
    - [ ] 필터: 부서, 신분, 학적상태, 재직상태

#### 수정/보완 필요
- **로그인 처리 로직** (백엔드)
  - [ ] ERP 연동 데이터 View 테이블 구조 설계

## 5. 심사 및 평가 관리

### 5.1 관리자 화면
#### 기존 기능 확인 필요
- **심사 기준 등록 화면**
  - 현재 상태: 메뉴 존재 (`evaluationCriteria`)
  - 확인 필요:
    - [ ] 평가표 유형 선택 (점수형/등급형/P/F형)
    - [ ] 점수형: 100점 만점 구조
    - [ ] P/F형: 합격/불합격 구조
    - [ ] 등급형: 후순위 (현재 제외 가능)

#### 신규 구현 필요
- **심사위원 배정 화면**
  - 현재 상태: 학위논문심사/학술지논문심사 화면 확인 필요
  - 필요 기능:
    - [ ] 학생별 심사 대상 목록
    - [ ] 심사위원 검색 모달
    - [ ] 심사위원장 1명 선택 (라디오 버튼)
    - [ ] 심사위원 N명 선택 (체크박스, 지도교수 포함)
    - [ ] 석사: 3명, 박사: 5명 기본 가이드
    - [ ] 일괄 저장

#### 수정 필요
- **심사 결과 조회/판정 화면**
  - [ ] 심사위원별 점수 입력 현황
  - [ ] 자동 판정 로직 (2/3 이상 기준 점수 통과)
  - [ ] 석사: 3명 중 2명 70점 이상
  - [ ] 박사: 5명 중 3명 70점 이상
  - [ ] 최종 판정 값: 통과/보완 후 통과/보류/불합격
  - [ ] 심사위원장 최종 승인 버튼

### 5.2 교수 화면
#### 신규 구현 필요
- **심사위원 점수 입력 화면**
  - 현재 상태: 미확인
  - 필요 기능:
    - [ ] 심사 대상 학생 목록 (본인이 심사위원인 경우)
    - [ ] 평가 항목별 점수 입력
    - [ ] 총점 자동 계산
    - [ ] 의견 입력
    - [ ] 제출 버튼

#### 추가 구현 (심사위원장)
- **최종 판정 화면**
  - [ ] 모든 심사위원 점수 확인
  - [ ] 자동 판정 결과 확인
  - [ ] 최종 판정 승인 버튼

### 5.3 학생 화면
#### 수정 필요
- **심사 결과 조회 화면**
  - [ ] 심사 진행 상태 표시
  - [ ] 심사위원 명단 (익명 or 공개 정책 확인 필요)
  - [ ] 최종 판정 결과 표시

## 8. 화면별 수정 우선순위

### 우선순위 1 (필수 구현)
1. **관리자 - 사용자 동기화 기능**
2. **관리자 - 교수 배정 화면**
3. **관리자 - 학생 단계 관리 (워크플로우 부여/이관)**
4. **관리자 - 심사위원 배정 UI**
5. **교수 - 심사 점수 입력 화면**

### 우선순위 2 (중요)
1. **관리자 - 권한 관리 화면**
2. **관리자 - 사용자 지원(대리 로그인)**
3. **관리자 - 심사 결과 판정 화면**
4. **학생 - 로그인 제어 및 안내 화면**
5. **교수 - 미배정 시 안내 화면**

### 우선순위 3 (개선)
1. **평가표 구성 (점수형/P/F형)**
2. **일정 관리 단순화**
3. **테이블 UI 개선**
