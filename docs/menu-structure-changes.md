## 목적

## 📋 주요 변경 사항 요약

### 1. 메뉴 구조 재편
**변경 전 (기존 문서 기준)**:
```
관리자 화면
├── 대시보드
├── 논문 안내
├── ⭐ 학생 관리 (최상위 메뉴)
│   ├── 지도교수 배정
│   └── 학생 단계 관리
├── 논문 제출
├── 논문 지도
├── 논문 심사
└── 시스템 설정
```

**변경 후 (Excel 제안 기준)**:
```
관리자 화면
├── 대시보드
├── 논문 안내
├── 논문 제출
│   ├── 논문제출요건등록
│   ├── 논문 제출 (연구계획서/1차 보고서/최종논문)
│   └── 최종논문제목수정
├── 논문지도
│   ├── ⭐ 지도 학생 관리 (통합 메뉴)
│   ├── 주차별 논문지도 현황
│   ├── 논문 지도 현황
│   └── 실시간 지도 예약
├── 논문심사
│   ├── 심사위원 배정
│   ├── 심사 기준 등록
│   ├── ⭐ 심사 단계 등록 (워크플로우 조립)
│   ├── 학위 논문 심사
│   └── 학술지 논문 심사
└── 시스템설정
    ├── 사용자관리 (대리로그인)
    ├── 권한관리
    ├── 일정관리
    └── 안내문관리
```

### 2. 핵심 차이점
1. **"학생 관리" 최상위 메뉴 제거**
   - **기존**: 독립된 최상위 메뉴
   - **Excel 제안**: "논문지도" > "지도 학생 관리"로 통합
   - **통합 기능**: 지도교수 배정 + 학생 지도 단계 변경(이관)

2. **논문 제출 메뉴 역할 명확화**
   - **교수 화면**: "논문 제출" 메뉴 **제거** (교수는 제출 주체가 아님)
   - **학생 화면**: "논문 제출" 메뉴 유지
   - **공통**: "최종논문제목수정" (교수, 학생, 관리자 모두 접근)

3. **심사 단계 등록 개념**
   - **Excel 제안**: "논문심사" > "심사 단계 등록" (워크플로우 조립)
   - **기존 문서**: "시스템설정" > "논문 지도 단계 설정" (학위별 단계 템플릿)
   - **확인 필요**: 두 개념이 다른지, 하나로 통합할지 검토

4. **사용자 관리 메뉴명**
   - **Excel 제안**: "사용자관리" (대리로그인 기능)
   - **기존 문서**: "사용자 지원"

### 3. 논문 제출
#### 3.1 논문제출요건등록
- **상태**: 변경 없음
- **접근 권한**: 관리자(O)
- **파일**: `admin/admin-requirement-management.html`

#### 3.2 ⭐ 논문 제출 (연구계획서 / 1차 보고서 / 최종논문)
- **상태**: 🔴 **중요 확인 필요**
- **접근 권한**:
  - 교수(X) - 교수는 제출 주체가 아님
  - 학생(O) - 제출 및 제출 결과 조회
  - 관리자(X)
- **비고**:
  - 교수는 '제출' 주체가 아니므로 교수용 화면에서는 '제출' 메뉴 삭제
  - 교수는 지도와 심사만 수행
  - 학생은 제출 목록에서 심사결과 조회
  - **(*) 확인 필요**: 학생이 심사 요청 후, 심사 요청된 내용의 수정 - 변경 가능여부

#### 3.3 최종논문제목수정
- **상태**: ✏️ **수정**
- **접근 권한**: 교수(O), 학생(O), 관리자(O)
- **파일**: `admin/admin-thesis-title.html`, `professor/professor-title-management.html`, `student/student-title-management.html`
- **비고**:
  - 학생 - 관리자 동일한 화면으로 구성 (학생도 목록 + 상세 화면으로 구성)
  - 학생만 보는 화면이라면 논문제목도 '제출'로 봐야하는 것 아닌지 확인
  - 제출 - 심사가 동시에 있을 경우, 혼동 우려 있음

## 프로토타입 우선순위 (Excel 기준 재조정)

### P1 (필수 - 1차 구현)
1. **⭐ 지도 학생 관리** (관리자)
   - 지도교수 배정
   - 학생 지도 단계 변경(이관)
   - 연구계획서 보기 모달
   - 교수 검색 모달

2. **심사위원 배정 UI** (관리자)
   - 심사위원 검색 모달
   - 배정 처리

3. **학위 논문 심사** (교수)
   - 심사 점수 입력
   - 평가 항목 입력

4. **사용자 상태 안내 화면** (교수/학생)
   - 미배정 시 안내 메시지

### P2 (중요 - 2차 구현)
1. **권한 관리 UI** (관리자)
   - 메뉴 접근 권한 매트릭스

2. **사용자관리 (대리 로그인)** (관리자)
   - 사용자 검색, 화면 전환

3. **심사 결과 판정** (관리자)
   - 자동 판정 로직

4. **메뉴 동적 제어** (교수/학생)
   - 상태별 메뉴 노출/숨김
