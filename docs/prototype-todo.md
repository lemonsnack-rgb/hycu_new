## Phase 1: 핵심 화면 구현 (3주)

### [P1-1] 관리자 - 지도교수 배정 화면
**예상 소요**: 3일

**메뉴명**: 지도교수 배정 (기존: 연구계획서 관리)
**프로세스**: 학생이 연구계획서 제출 → 관리자가 연구계획서를 확인하고 지도교수 매핑

#### 화면 구성
- [ ] 학생 목록 테이블 HTML 구조 작성
  - 컬럼: 체크박스, 학번, 이름, 학과, 연구계획서 제출일, 배정 상태, 연구계획서 보기, 배정 버튼
  - 필터: 학과, 배정 상태
  - 검색: 학번/이름
  - 예상: 0.5일

- [ ] 연구계획서 보기 모달
  - 학생 정보 (학번, 이름, 학과, 연락처)
  - 연구 주제, 연구 목적, 연구 방법 등
  - 제출일
  - 예상: 0.5일

- [ ] Mock 데이터 생성 (`mockData.js`)
  - 학생 20명 (배정 대기 10명, 배정 완료 10명)
  - 연구계획서 데이터 (주제, 목적, 방법)
  - 교수 10명 (소속 학과 5개, 전임/비전임 구분)
  - 예상: 0.5일

- [ ] 주지도교수 검색 모달 구현
  - 라디오 버튼으로 1명 선택
  - 검색: 이름/사번
  - 필터: 소속 학과, 타과 포함 체크박스
  - 정렬: 전임교원 우선 (배지 표시)
  - 예상: 0.5일

- [ ] 공동지도교수 검색 모달 구현
  - 체크박스로 다중 선택
  - 동일 검색 UI
  - 예상: 0.5일

- [ ] 배정 처리 로직
  - 선택한 교수 정보 표시
  - 배정 완료 버튼 클릭 시 Mock 데이터 업데이트
  - 성공 토스트 메시지
  - 테이블 재렌더링
  - 예상: 0.5일
