## 📋 핵심 변경 사항 (Excel 기준)

### 1. 메뉴 구조 재편
- **"학생 관리" 최상위 메뉴 제거**
  - **기존 문서**: 독립된 최상위 메뉴
  - **Excel 제안**: "논문지도" > "지도 학생 관리"로 통합
  - **통합 기능**: 지도교수 배정 + 학생 지도 단계 변경(이관)

### 2. 논문 제출 메뉴 역할 명확화
- **교수 화면**: "논문 제출" 메뉴 **제거** (교수는 제출 주체가 아님)
- **학생 화면**: "논문 제출" 메뉴 유지 (연구계획서/1차 보고서/최종논문)
- **공통**: "최종논문제목수정" (교수, 학생, 관리자 모두 접근)

### 3. 심사 단계 등록 개념
- **Excel 제안**: "논문심사" > "심사 단계 등록" (워크플로우 조립)
- **기존 문서**: "시스템설정" > "논문 지도 단계 설정" (학위별 단계 템플릿)
- **확인 필요**: 두 개념이 다른지, 하나로 통합할지 검토

## 🔴 확인 필요 사항 (고객 확인 필수)

### 1. 심사 단계 등록 개념
- **질문**: "심사 단계 등록" (Excel)과 "논문 지도 단계 설정" (기존 문서)이 같은 기능인가요?
  - **심사 단계 등록**: 워크플로우 조립 (본심사, 예비심사 등)
  - **논문 지도 단계 설정**: 학위별 단계 템플릿 (석사 5단계, 박사 7단계)
- **확인**: 두 개념이 다르면 둘 다 구현해야 함

### 2. 학생 심사 요청 후 수정 가능 여부
- **질문**: 학생이 심사 요청 후, 심사 요청된 내용의 수정 - 변경이 가능한가요?

### 3. 관리자 실시간 지도 예약 화면
- **질문**: 실시간 지도 예약을 관리자가 보는 화면이 필요한가요?
- **관리자 역할**: 현황 조회만? 예약 변경 가능?

### 4. 안내문관리 메뉴
- **질문**: 안내문관리는 콘텐츠 관리 기능으로 보면 되나요?
- **범위**: 프로토타입에 포함할지 여부

### 5. 교수 화면 "논문 제출" 메뉴
- **Excel 제안**: 교수 화면에서 "논문 제출" 메뉴 제거
- **확인**: 교수는 제출 주체가 아니므로 메뉴 혼동 방지 - 동의하시나요?

## 주요 메뉴 구조 (Excel 기준)

### 관리자 화면
```
관리자 화면
├── 대시보드
├── 논문 안내
│   ├── 공지사항
│   ├── 자료실
│   ├── 연구윤리
│   ├── 논문일정
│   └── 논문지도절차
├── 논문 제출
│   ├── 논문제출요건등록
│   ├── 논문 제출 (연구계획서/1차 보고서/최종논문)
│   └── 최종논문제목수정
├── 논문지도
│   ├── ⭐ 지도 학생 관리 (통합 메뉴)
│   ├── 주차별 논문지도 현황
│   ├── 논문 지도 현황
│   └── 실시간 지도 예약
├── 논문심사
│   ├── 심사위원 배정
│   ├── 심사 기준 등록
│   ├── ⭐ 심사 단계 등록 (워크플로우 조립)
│   ├── 학위 논문 심사
│   └── 학술지 논문 심사
└── 시스템설정
    ├── 사용자관리 (대리로그인)
    ├── 권한관리
    ├── 일정관리
    └── 안내문관리
```

### 교수 화면
```
교수 화면
├── 대시보드
├── 논문 안내
├── 논문 제출 ❌ (제거)
│   └── 최종논문제목수정 (유지)
├── 논문지도
│   ├── 주차별 논문지도 현황
│   ├── 논문 지도 현황
│   └── 실시간 지도 예약
└── 논문심사
    ├── 학위 논문 심사
    └── 학술지 논문 심사
```

### 학생 화면
```
학생 화면
├── 대시보드
├── 논문 안내
├── 논문 제출
│   ├── 논문 제출 (연구계획서/1차 보고서/최종논문)
│   └── 최종논문제목수정
├── 논문지도
│   ├── 주차별 논문지도 현황
│   ├── 논문 지도 현황
│   └── 실시간 지도 예약
└── (심사 메뉴 없음 - 제출한 목록에서 심사결과 조회)
```

## 프로토타입 우선순위 (Excel 기준)

### P1 (필수 - 1차 구현)
1. **⭐ 지도 학생 관리** (관리자)
   - 지도교수 배정
   - 학생 지도 단계 변경(이관)
   - 연구계획서 보기 모달
   - 교수 검색 모달

2. **심사위원 배정 UI** (관리자)
   - 심사위원 검색 모달
   - 배정 처리

3. **학위 논문 심사** (교수)
   - 심사 점수 입력
   - 평가 항목 입력

4. **사용자 상태 안내 화면** (교수/학생)
   - 미배정 시 안내 메시지

### P2 (중요 - 2차 구현)
1. **권한 관리 UI** (관리자)
2. **사용자관리 (대리 로그인)** (관리자)
3. **심사 결과 판정** (관리자)
4. **메뉴 동적 제어** (교수/학생)
5. **일정 조회 화면** (교수/학생)

### P3 (개선 - 3차 구현)
1. **평가표 구성 UI** (관리자)
2. **심사 단계 등록** (관리자) - **개념 확인 후 구현**
3. **테이블 UI 개선** (공통)
4. **로딩 상태 표시** (공통)
