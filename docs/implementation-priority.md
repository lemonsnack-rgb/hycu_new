# 논문 지도 시스템 우선순위별 구현 계획

## 작성 일자
2025-12-08

## 목적
회의 결과 및 수정 대상 식별을 기반으로 우선순위별 구현 계획 수립

---

## 우선순위 분류 기준

### Priority 1 (P1) - 필수/긴급
시스템 핵심 기능, 없으면 운영 불가능한 기능

### Priority 2 (P2) - 중요
사용자 편의성 및 업무 효율성에 직접적 영향

### Priority 3 (P3) - 개선
있으면 좋지만 당장 없어도 운영 가능한 기능

---

## P1: 필수 구현 항목 (8개 작업)

### 1. [P1-1] ERP 사용자 동기화 (백엔드 + 프론트)
**담당 화면**: 관리자

**작업 내용**:
- 백엔드: ERP View 테이블 연동 API 구현
- 백엔드: 배치 동기화 스케줄러 구현 (일 1회 또는 시간 단위)
- 프론트: 사용자 관리 화면에 "동기화" 버튼 추가
- 프론트: 동기화 진행 상태 및 결과 표시
- 프론트: 동기화 이력 조회 기능

**선행 조건**:
- ERP View 테이블 스키마 확정
- SSO 연동 완료

**예상 소요**: 5일

---

### 2. [P1-2] 로그인 제어 로직 (백엔드 + 프론트)
**담당 화면**: 전체 (학생/교수/관리자)

**작업 내용**:
- 백엔드: SSO 연동 후 권한 체크 미들웨어
- 백엔드: 학생 - 논문과정 선택 여부 체크
- 백엔드: 특수 학적/재직 상태 접근 제어 (정책 확정 후)
- 프론트: 학생 - 논문과정 미선택 시 안내 페이지
- 프론트: 교수 - 미배정 시 안내 페이지
- 프론트: 특수 상태 시 읽기전용 모드 또는 차단

**선행 조건**:
- SSO 연동 방식 확정
- 특수 학적 상태 접근 제어 정책 확정

**예상 소요**: 3일

---

### 3. [P1-3] 교수 배정 화면 (관리자)

- 화면: 교수 배정 모달 (주지도교수 1명, 공동지도교수 N명)
- 기능: 교수 검색 (이름, 소속, 전공)
- 기능: 필터 (소속 학과 우선, 타과 포함)
- 기능: 배정 저장
- 백엔드: 교수 배정 API

**선행 조건**:
- 연구계획서 제출 화면 존재 여부 확인
- 교수 정보 테이블 구조 확인

**예상 소요**: 4일

### 4. [P1-4] 워크플로우 부여 및 단계 이관 (관리자)
