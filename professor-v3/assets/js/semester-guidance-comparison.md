# 학기별 논문지도 V1 vs V2 상세 비교

## 🎯 핵심 차이점 요약

| 구분 | V1 | V2 |
|------|----|----|
| **실적 입력 방식** | 모달 팝업 | 댓글 방식 인라인 |
| **화면 구성** | 등록/조회 분리 | 통합 화면 |
| **학기 선택** | ❌ 없음 | ✅ 드롭다운 (5년) |

### 실적 입력 과정
```
사용자 행동 흐름:

1. 주차 클릭 → 아코디언 펼침
   ┌─────────────────────────────────┐
   │ 1주차 │ 연구방법론 개요           │ ▲
   ├─────────────────────────────────┤
   │ 계획 내용: ...                   │
   │ [실적 추가] ← 버튼 클릭           │
   └─────────────────────────────────┘

2. 모달 팝업 오픈
   ┌──────────── 모달 ────────────┐
   │  실적 추가                    │
   │  ─────────────────────       │
   │  실행일: [____]               │
   │  지도방식: [▼]                │
   │  실행내용: [_______]          │
   │  교수의견: [_______]          │
   │                               │
   │  [취소]  [저장] ← 클릭        │
   └──────────────────────────────┘

3. 저장 → 모달 닫힌 → 목록 새로고침

4. 다음 실적 추가하려면 1번부터 다시 반복
```

### 단점
- ❌ 모달 반복 오픈/클릭 피로도
- ❌ 저장 후 화면 새로고침 딜레이
- ❌ 연속 입력이 어려움
- ❌ 과거 학기 조회 불가
- ❌ 현재 학기만 보임

## 💡 핵심 개선 포인트

### 1. 댓글 방식 (Comment-Style Input)
**V1 문제점:**
- 모달 오픈 → 입력 → 저장 → 닫기 → 반복
- 화면 맥락 전환으로 집중력 분산

**V2 해결책:**
- 실적 목록과 입력폼을 같은 위치에 배치
- SNS 댓글처럼 자연스러운 흐름
- 화면 전환 없이 연속 입력

### 2. 학기 선택 (Semester Selector)
**V1 문제점:**
- 현재 학기만 보임
- 과거 데이터 조회 불가

**V2 해결책:**
- 최근 5년간 모든 학기 선택 가능
- 계획 존재 여부 표시 (✓)
- 드롭다운으로 즉시 전환

### 3. 통합 화면 (Unified Interface)
**V1 문제점:**
- 등록 화면과 조회 화면 분리
- 사용자 혼란 가능

**V2 해결책:**
- 등록 + 조회를 하나의 화면으로
- 계획 없음 → 등록 버튼 표시
- 계획 있음 → 주차 카드 + 입력폼 표시
